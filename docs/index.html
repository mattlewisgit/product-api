User Story

Title: Product and Order API

As a user, I want to create, view, update, and delete products, and create customer orders containing products, so that I can manage a simple product ordering system through an API.

Acceptance Criteria

  • Users can create a new product using POST /api/products
  • Users can retrieve all products using GET /api/products
  • Users can retrieve a single product using GET /api/products/{id}
  • Users can update an existing product using PUT /api/products/{id}
  • Users can delete a product using DELETE /api/products/{id}
  • Users can create an order containing one or more product IDs using POST /api/orders
  • Users can retrieve all orders using GET /api/orders
  • Users can retrieve a single order using GET /api/orders/{id}
  • Users can delete an order using DELETE /api/orders/{id}
  • The API should return appropriate responses for invalid or missing data

Testing Task

Use Swagger UI to test the Product API endpoints. Create test products, update them, retrieve them, delete them, and create customer orders using existing product IDs.
